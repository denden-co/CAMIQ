System Architecture

Frontend — Next.js on Vercel

A modern React-based frontend using Next.js App Router with internationalisation (i18n) support. The UI features a country/election selector that dynamically configures the entire platform for any nation's political parties and electoral system. Three main tabs: Social Data Analysis, Single Text Analysis, and Voter Persona Generator. Deployed to Vercel for fast global CDN delivery.

Next.js 14+ React 18 TypeScript Tailwind CSS Recharts shadcn/ui next-intl (i18n)

Backend — Python FastAPI

A Python-based API server with country-agnostic ML models. The ensemble sentiment classifier uses multilingual transformers (XLM-RoBERTa) for global language support alongside English-specific models (BERTweet). BERTopic and LDA run topic extraction in any language. ANOVA and statistical analysis adapt to any party structure. Deployed as a separate service (Railway, Render, or AWS Lambda) for persistent GPU/compute.

FastAPI Python 3.11+ Transformers (HuggingFace) XLM-RoBERTa (multilingual) scikit-learn BERTopic pandas / numpy scipy (ANOVA)

LLM / AI API Integration Layer

Universal API abstraction layer supporting all major LLM and AI providers. Users can plug in their own API key for any supported provider. The system auto-routes to the selected provider for the AI Strategic Advisor, Single Text Analysis, Voter Persona Generator, and the new AI-assisted data cleaning features.

Google Gemini API OpenAI API Anthropic Claude API Deepseek API Mistral API Cohere API Meta Llama (via Together/Groq) HuggingFace Inference API Custom OpenAI-compatible endpoint

Data Cleaning & Preprocessing Engine

Built-in data pipeline that cleans raw social media data before analysis. Handles messy real-world CSV files with missing values, duplicates, encoding issues, and noisy text. Users get a visual summary of what was cleaned and can adjust settings.

pandas profiling Text normalisation (emoji, URL, @mention handling) Duplicate detection Missing value imputation Language filtering Bot/spam detection Encoding fix (UTF-8, Latin-1)

Multi-Algorithm Analysis Engine

CampaignIQ goes beyond the thesis models. Users can choose from a full library of ML, deep learning, NLP, and statistical algorithms — or let the system recommend the best approach for their data. Results from different models can be compared side-by-side.

Data & Storage

Supabase for user auth, API key storage (encrypted), and analysis history. File uploads handled via temporary server-side processing. No long-term storage of uploaded CSV data for privacy reasons.

Global Multi-Country Design

How It Works for Any Country

CampaignIQ is built country-agnostic from the ground up. The thesis was proven on the UK 2024 General Election, but the platform is designed so that a user in Brazil, India, Nigeria, France, or any other democracy can use it for their own elections. Here's how:

Country/Election Configuration: When a user starts a new analysis, they first select their country and election (or create a custom one). This loads the relevant parties, electoral system type, and political context. Pre-built configs ship for major democracies; users can also define custom elections with their own party names and colours.

Multilingual Sentiment: The ML pipeline uses XLM-RoBERTa (a multilingual transformer supporting 100+ languages) as the primary classifier, with BERTweet as a specialist for English tweets. This means sentiment classification works in Spanish, Portuguese, Hindi, French, Yoruba, Arabic, and more.

Electoral System Awareness: The diagnostic framework adapts to different electoral systems — FPTP (UK, USA, India), proportional representation (Germany, Netherlands), mixed systems (Japan, New Zealand), and others. The disclaimer layer adjusts based on what kind of prediction is feasible under each system.

Core Features

🌍

1. Country & Election Setup

Select from pre-built country profiles (UK, USA, India, Brazil, France, Nigeria, Germany, etc.) or create a custom election. Configures parties, colours, electoral system type (FPTP, PR, mixed), and political context automatically.

🧹

2. Data Cleaning Studio

Automatic and manual data cleaning before analysis. Removes duplicates, handles missing values, normalises text (URLs, emojis, @mentions), detects bots/spam, fixes encoding. Visual before/after summary with adjustable settings. Supports CSV, Excel, JSON, and API imports.

📊

3. Social Data Analysis Portal

Upload cleaned data in any language. Select parties and choose algorithms — from the thesis ensemble (XLM-RoBERTa + BERTweet + ANOVA) to classic ML (SVM, Random Forest, XGBoost) to deep learning (CNN, LSTM). Compare model results side-by-side. Output: vote share predictions, sentiment dashboards, topic breakdowns.

🔬

4. Algorithm Lab

Choose from 20+ algorithms across sentiment analysis (VADER, TextBlob, transformers), classification (Logistic Regression, SVM, Random Forest, XGBoost, CNN, LSTM), topic modelling (BERTopic, LDA, NMF), clustering (K-Means, DBSCAN), and statistics (ANOVA, Chi-Square, regression). Auto-recommend the best approach or pick manually.

🔍

5. Single Text Analysis

Enter any political text in any language. Detects emotions (joy, sadness, anger, surprise, fear) with radar chart. Identifies political topics as tags — automatically adapted to the selected country's issues. Powered by any connected LLM provider.

👤

6. Voter Persona Generator

Input demographics, primary concerns, and core values for any country. LLM generates culturally relevant voter personas with backstory, concerns, political leanings, and how to appeal to them in that specific national context.

🧠

7. AI Strategic Advisor

Country-aware strategic campaign recommendations. Understands the difference between advising on a UK marginal seat, a US swing state, or a Brazilian state election. Counter-messaging and boost strategies tailored to the political context.

⚖️

8. Bias & Fairness Audit

Model objectivity checks using Gini coefficient across all parties in the configured election. Ensures fair and transparent analysis regardless of country or number of parties.

⚙️

9. API Settings Panel

Universal LLM provider selector: Google Gemini, OpenAI, Anthropic Claude, Deepseek, Mistral, Cohere, Meta Llama (via Together/Groq), HuggingFace, or any OpenAI-compatible endpoint. Save API keys securely. Test connection before saving.

Risks & Mitigations

Risk Severity Mitigation
ML model size too large for serverless (RoBERTa ~500MB) High Use model distillation, host on Railway/Render with persistent compute, or use HuggingFace Inference API
Cold start latency on Python API Medium Keep-alive pings, model caching, consider dedicated VM for API
LLM API costs for users Low Users supply their own API keys; no cost to CampaignIQ
CSV processing timeout on large files Medium Batch processing with progress bar, max 10,000 rows per upload, async processing with polling
Twitter/X API access changes Medium App works with pre-collected CSV data; not dependent on live Twitter API
Accuracy expectations vs thesis limitations Medium Clear disclaimers on dashboard: "measurement tool, not a predictor" — aligned with thesis conclusions
Multilingual sentiment accuracy varies by language High XLM-RoBERTa performs best on high-resource languages (English, French, Spanish, German). Show confidence badge per language. Allow community fine-tuning in future
Electoral system differences affect prediction validity Medium Dynamic disclaimers per electoral system type. PR systems get "indicative only" labels. FPTP gets the full diagnostic framework from thesis
Maintaining country configs as politics change Low Users can create custom elections with any parties. Pre-built configs are starting points, not hard dependencies. Community contribution model for updates
